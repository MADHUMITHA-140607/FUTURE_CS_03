# API Security Risk Analysis

## 📌 Project Overview

This project was completed as part of the Future Interns Cyber Security Internship Task 3.

The objective of this project was to perform a read-only API Security Risk Analysis on a public API, identify potential security risks, assess business impact, and recommend remediation measures.

---

## 🎯 Objectives

- Analyze a public API using Postman
- Inspect API responses and exposed data
- Identify API security risks
- Assess business impact
- Classify risk severity levels
- Recommend security improvements

---

## 🚨 Risks Identified

### Risk 1 – Open / Unauthenticated Endpoint
Severity: Medium

### Risk 2 – Excessive Data Exposure
Severity: Medium

### Risk 3 – Potential Missing Rate Limiting
Severity: Low

### Risk 4 – Missing Authentication Controls
Severity: Medium

---

## 🛡️ Recommendations

- Implement JWT/OAuth authentication
- Minimize exposed response data
- Apply rate limiting controls
- Restrict access to sensitive endpoints
- Follow OWASP API Security best practices
